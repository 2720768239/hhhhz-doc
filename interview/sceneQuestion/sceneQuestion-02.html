如何解决页面请求接口大规模并发问题

大文件上传

目录

大文件上传 ​

背景 ​

例如：之前做 AI 产品方面，会涉及到用户自定义模型（1G 以上）

经常会遇到一些问题：

  • 网络断开之后，之前传的没了
  • 上传过程中，网络出了问题，或者浏览器崩溃了，结果都没了
  • 关机了，想接着传，结果得重新传

专业术语：

  • 断点续传
  • 断开重连重传
  • 分片（切片）上传

方案 ​

  • 前端切片 chunk 1024M（1048576K），500K， const size = 1048576 / 500
  • 将切片传递给后端，切片需要取名：hash，index
  • 后端组成切片

落地 ​

html
<div>
  <input type="file" id="fileInput" />
  <button onclick="upload()">上传</button>
</div>
js
const CHUNK_SIZE = 5 * 1024 * 1024 // 每块大小为5M

function upload() {
  const file = document.getElementById('fileInput').files[0]
  if (!file) {
    alert('请选择文件')
    return
  }

  const totalChunks = Math.ceil(file.size / CHUNK_SIZE)
  let currentChunk = 0

  function uploadChunk() {
    if (currentChunk >= totalChunks) {
      console.log('Upload complete')
      return
    }

    const start = currentChunk * CHUNK_SIZE
    const end = Math.min(start + CHUNK_SIZE, file.size)
    const chunk = file.slice(start, end)

    const formData = new FormData()
    formData.append('file', chunk)
    formData.append('name', file.name)
    formData.append('chunkNumber', currentChunk + 1)
    formData.append('totalChunks', totalChunks)

    fetch('/upload', {
      method: 'POST',
      body: formData,
    })
      .then((response) => response.json())
      .then((data) => {
        if (data.success) {
          console.log('Chunk uploaded:', data)
          currentChunk++
          uploadChunk()
        } else {
          console.error('Error uploading chunk:', data.error)
        }
      })
      .catch((error) => {
        console.error('Error uploading chunk:', error)
      })

    uploadChunk()
  }

反思和优化 ​

  • 前端切片：主进程卡顿，用 web-worker 多线程切片，处理后交给主进程发送给后端
  • 切完后，将 blob，储存到 indexDB，下次用户再上传，直接从 indexDB 中读取，是否存在未完成的上传切片，存在的话，直接上传，不存在，重新上传
  • 可以引进 websocket，实时通知，和请求序列的控制 wss
  • 整体说说我主导的大文件上传器整体设计
    • 组件设计
    • props、事件、状态
    • 拖拽上传、多文件选择
    • 通用化不同文件的上传，上传统一协议
