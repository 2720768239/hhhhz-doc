作用域 ​

规定了变量能够被访问的“范围”，离开了范围，变量则不能访问

局部作用域 ​

函数作用域和块作用域

  1. 函数作用域 只能函数内部访问，外部不能访问
js
function getNum() {
  const num = 10
}
coonsole.log(num) // 报错
1
2
3
4
  • 函数内部声明的变量，外部无法被访问
  • 函数内部的参数也是函数内部的局部变量
  • 不同函数内部声明的变量无法互相访问
  • 函数执行完后，函数内部的变量实际被清空了
  1. 块作用 在{}内部声明的变量，外部无法访问
js
for (let i = 0; i < 10; i++) {
  console.log(i)
}
console.log(i) // 报错
1
2
3
4
  • let声明的变量会产生块级作用域， var不会产生块级作用域
  • const声明的常量也是块级作用域
  • 不用代码之间的变量无法互相访问
  • 建议使用let和const

全局作用域 ​

<script>和.js文件的最外层就是全局作用域，全局作用域声明的变量可以在任何地方访问

js
const num = 10
function getNum() {
  console.log(num)
}
getNum() // 10

4

注意：

  1. 为 window 对象添加的属性，也是全局
  2. 函数未使用任何关键字声明，默认也是全局
  3. 少用全局变量，防止变量污染

作用域链 ​

本质是底层的变量查找机制

  • 函数被执行时，会优先查找当前函数作用域中，查找变量
  • 如果当前作用域没有，会向上级作用域查找，直到全局作用域
  1. 嵌套关系的作用域串联成了作用域链
  2. 相同作用域中按从小到大的规则查找变量
  3. 子作用域能够访问父作用域，父无法访问子作用域

JS 垃圾回收机制 ​

垃圾回收机制(Garbage Collection)，简称 GC。JS 内存的分配和回收都是自动完成的，内存在不使用的时候会被垃圾回收器自动回收

内存的生命周期

  1. 内存分配。当声明变量、函数、对象的时候，系统会自动分配内存
  2. 内存使用。即读写内存，也就是使用变量、函数等
  3. 内存回收。使用完毕，由垃圾回收器自动回收不再使用的内存
js
//  内存分配
const age = 19
const obj = { name: 'hao' }
function getName() {
  return 'hao'
}

说明

  1. 全局变量一般不回收，在关闭页面时回收
  2. 一般情况下局部变量，函数执行完毕后，局部变量会自动回收

内存泄漏：程序中分配的内存，由于某种原因程序未释放或无法释放，叫内存泄漏

算法说明：

  1. 栈：由操作系统自动分配释放函数的参数值、局部变量等，基本数据类型放到栈中
  2. 堆：一般由程序员分配释放，若不释放，则由垃圾回收机制回收。复杂数据类型放到堆里

常见浏览器垃圾回收算法：引用计数法、标记清除法

  • 引用计数法：一个对象是否有指向它的引用，没有引用了就回收
js
const arr = [1, 2, 3]
arr = null // 回收

let person = { age: 10 }
let p = person
person = 1
p = null // 回收

  • 标记清除法：从根节点出发，标记所有可达的对象，其余没有标记的对象就是垃圾，回收
  1. 将“不再使用的对象”定义为“无法达到的对象”
  2. 从根部（js 的全局对象）出发，定时扫描内存中的对象。凡是能从根部到达的对象，都还是需要使用的
  3. 无法从根部出发触及到的对象会被标记为不再使用，稍后进行回收

核心思路：从根出发，无法触及的对象就是垃圾

变量提升 ​

变量提升是 JavaScript 中比较奇怪的现象，它允许变量在声明之前即被访问（仅存在 var 声明的变量）

代码执行之前，会先进行预解析，将 var 声明的变量提升到当前作用域的最前面

// 2.只提升声明，不赋值

实例成员和静态成员 ​

实例成员：通过构造函数创建的对象称为实例对象，实例对象中的属性和方法称为实例成员（实例属性和实例方法）

说明：

  1. 为构造函数传入参数，创建结构相同但值不同的对象
  2. 构造函数创建的实例对象彼此独立互补影响

静态成员：构造函数的属性和方法称为静态成员（静态属性和静态方法）

说明：

  1. 静态成员只能构造函数来访问
  2. 静态方法中的 this 指向构造函数
js
function Person(name, age) {
  // 实例成员
}
// 静态属性
Person.name = 'hz'
Person.age = 19
// 静态方法
Person.say = function () {
  console.log('hello')
  // this.指向Person
  console.log(this.name) // hz
}

深入面向对象 ​

这里主要是原型链的方式

编程思想 ​

  • 面向过程：分析出解决问题的步骤，然后用函数把这些步骤一步一步实现，使用时再依次调用

  • 面向对象：把事务分解成一个个对象，然后对象之间分工与合作 以对象功能划分，不是步骤

面向对象 ​

  • 每一个对象都是功能中心，具有明确分工
  • 具有灵活、代码可复用、容易维护和开发的优点，适合多人合作的大型项目
  • 特性：封装性、继承性、多态性

构造函数 ​

  • 封装是面向对象思想的核心，js 中通过构造函数实现封装
  • 构造函数实例创建的对象彼此独立，互不影响
  • 存在内存浪费问题

原型 ​

解决构造函数导致的内存浪费问题

原型 ​

  • 是什么：是一个对象，prototype 也称为原型对象

  • 作用：共享方法

  • 构造函数和原型里的 this 指向：实例化的对象

  • 构造函数通过原型分配的函数是所有对象所共享的

  • 每一个构造函数都有 prototype 属性，指向另一个对象，也叫原型对象

  • 对象可以挂载函数，对象实例化不会多次创建原型上的函数，节约内存

js
function Person(name, age) {
  this.name = name
  this.age = age
  // 不写这里
  // function say() {
  //   console.log('hello')
  // }
}
Person.prototype.say = function () {
  console.log('hello')
}
// 不管Person实例化多少个对象，都共同调用原理里的say方法，解决了内存浪费的问题

  }

性能优化 ​

防抖 debounce ​

单位时间内，频繁触发事件，只执行最后一次

使用场景：

输入框输入，只需最后一次输入输入完

  • 搜索框输入。
  • 手机号、邮箱验证输入检测

实现方式：

  1. lodash
  2. 自己实现
js
// 核心是利用setTimeout定时器实现
// 1. 声明定时器变量
// 2. 事件触发，先判断是否有定时器，如果有，先清除定时器
// 3. 如果没有定时器，则开启定时器，存入定时器变量
// 4. 定时器里写函数调用
function debouce(fn, t) {
  let timer = null
  // return 一个匿名函数
  return function () {
    if (timer) clearTimeout(timer)
    timer = setTimeout(() => {
      fn()
    }, t)
  }
}

节流 throttle ​

单位时间内，频繁触发事件，只触发一次

使用场景：

  • 高频事件：鼠标移动mousemove、页面缩放resize、滚动条滚动scroll

手写核心思路：利用定时器 setTimeout 实现

  1. 声明定时器变量
  2. 事件触发，判断是否有定时器，如果有，不开启新定时器
  3. 如果没有，则存到变量里。定时器里调用需要执行的函数，定时器里把定时器变量置为 null
js
function throttle(fn, t) {
  let timer = null
  return function () {
    if (timer) return
    timer = setTimeout(() => {
      fn()
      // 在setTimeout中，无法删除定时器，因为定时器还在运作，所以使用timer = null
      timer = null
    }, t)
  }
}
