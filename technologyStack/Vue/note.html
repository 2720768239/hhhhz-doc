setup ​

setup 是 vue3 中的一个新的配置项，值是一个函数。数据、方法、计算属性、监视等等，均配置在 setup 中

  • setup 函数返回的对象中的内容，可以直接在模版中使用
  • setup 函数访问 this 时 undefined
  • setup 函数会在 beforeCreate 前执行
typescript
<script lang="ts">
  export default {
    name: 'Person',
    setup() {
      const name = '张三'
      function changeName() {
        name = '李四' // name改了，但不是响应式的，所以这里页面上不会改为李四
      }
      // 将数据、方法return出去，模版中才可以使用
      return {
        name,
        changeName,
      }

      // setup的返回值也可以是个渲染函数
      // return () => h('h1', 'hello')
    },
  }
<script>

响应式 ​

ref、reactive

ref ​

定义基本类型、对象类型的数据

  • 作用：定义响应式变量
  • 语法：let xxx = ref(值)
  • 返回值：一个RefImpl的实例对象，简称ref对象或ref，ref对象的value属性是响应式的
  • 在 js 中操作数据需要以xxx.value的方式，但模版中不需要.value，直接使用即可
  • 对于let name = ref('张三')，name不是响应式的，name.value是响应式的
typescript
const name = ref('张三')
name.value = '李四'
1
2

reactive ​

定义对象类型数据（只能）

  • 作用：定义响应式对象
  • 语法：let xxx = reactive(值)
  • 返回值：一个Proxy的实例对象，简称proxy对象或proxy，proxy对象的属性是响应式的
  • 在 js 中操作数据不需要.value，直接操作即可
typescript
let car = reactive({ brand: 'benz', price: 40 })
car.price = 50

const person = reactive('张三') // 警告：value cannot be made reactive: "张三"
1
2

二者对比 ​

  1. 宏观角度
  • ref：基本是护具类型、对象数据类型
  • reactive：对象数据类型
  1. 区别
  • ref：变量需要使用.value（可以使用volar插件自动添加.value
  • reactive：变量不需要使用.value；重新分配一个新对象，会失去响应式，可以使用 Object.assign 去整体替换
typescript
let car = reactive({ brand: 'benz', price: 40 })
// 失去响应式
car = { brand: 'bmw', price: 50 }
// 失去响应式
// car = reactive({ brand: 'bmw', price: 50 })

// 需要使用Object.assign，才不会失去响应式
Object.assign(car, { brand: 'bmw', price: 50 })

  1. 使用规则
  • 一个基本数据类型的响应式数据，使用ref
  • 一个对象类型，但层级不深，ref、reactive都可
  • 一个对象类型，层级较深，使用reactive

let { name, age } = person
name = '李四' // name改了，但不是响应式的，所以这里页面上不会改为李四

computed ​

作用：根据已有数据计算出新数据

typescript
// 只读
const fullName = computed(() => {
  return person.firstName + '-' + person.LastName
})

// 可读可写
const fullName = computed({
  get() {
    return person.firstName + '-' + person.LastName
  },
  set(value) {
    const [str1, str2] = value.split('-')
    person.firstName = str1
    person.LastName = str2
  },
})

作用：监听数据的变化 特点：vue3 中只能监听四种数据

  • ref
  • reactive
  • 函数返回一个值(getter函数)
  • 一个包含上述内容的数组

情况 1：监听 ref 定义的基本类型 ​

vue

情况 2：监听 ref 定义的对象类型数据 ​

直接写数据名，监听的是对象的地址值，若想监听对象内部的数据，需要开启深度监听deep: true

  • 若修改的 ref 定义的对象中的属性，newValue 和 oldValue 都是新值，因为它们同一个对象
  • 若修改整个 ref 对象，newValue 是新值，oldValue 是旧值，因为不是同一个对象了
typescript
const person = ref({ name: '张三', age: 19 })
watch(person, (newValue, oldValue) => {}, { deep: true })

情况 3： 监听 reactive 定义的对象类型数据 ​

隐式深度监听，默认开启深度监听，即 deep 默认是 true，且不能关闭

情况 4： 监听 ref 或 reactive 定义的对象类型中的某个属性 ​

  • 若属性值不是对象，需要写成函数形式
  • 若属性值是对象，可以写成函数（更推荐）
typescript
const person = reactive({ name: '张三', age: 19, car: { c1: '车1', c2: '车2' } })
// 若想只监听person.name，其他不监听
// 会报警告
watch(person.name, (newValue, oldValue) => {})

// 正确写法：
watch(
  () => person.name,
  (newValue, oldValue) => {}
)

// 若想监听person.car，其他不监听
// 单独该person.car.c1或c2，能监听到，但是如果把整个person.car都改了，监听不到
watch(person.car, (newValue, oldValue) => {})
// 若想监听整个person.car:
watch(
  () => person.car,
  (newValue, oldValue) => {}
  {deep: true}
)

watchEffect ​

  • 作用：立即运行一个函数，同时响应式地追踪其依赖，并在依赖更改时重新执行该函数
  • watch 和 watchEffect 的区别：
  1. 都能监听响应式数据的变化，不同的是监听数据变化的方式不同
  2. watch 需要明确指出监听的数据
  3. watchEffect 不需要明确指出，会自动监听
typescript

生命周期 ​

生命周期整体分为四个阶段，分别是：创建、挂载、更新、销毁，每个阶段都有两个钩子，一前一后。

  • Vue2的生命周期

    创建阶段：beforeCreate、created

    挂载阶段：beforeMount、mounted

    更新阶段：beforeUpdate、updated

    销毁阶段：beforeDestroy、destroyed

  • Vue3的生命周期

    创建阶段：setup

    挂载阶段：onBeforeMount、onMounted

    更新阶段：onBeforeUpdate、onUpdated

    卸载阶段：onBeforeUnmount、onUnmounted

    vue
    <template>
  <div class="person">
    <h2>当前求和为：{{ sum }}</h2>
    <button @click="changeSum">点我sum+1</button>
  </div>
</template>

<!-- vue3写法 -->
<script lang="ts" setup name="Person">
import {
  ref,
  onBeforeMount,
  onMounted,
  onBeforeUpdate,
  onUpdated,
  onBeforeUnmount,
  onUnmounted,
} from 'vue'

// 数据
let sum = ref(0)
// 方法
function changeSum() {
  sum.value += 1
}
console.log('setup')
// 生命周期钩子
onBeforeMount(() => {
  console.log('挂载之前')
})
onMounted(() => {
  console.log('挂载完毕')
})
onBeforeUpdate(() => {
  console.log('更新之前')
})
onUpdated(() => {
  console.log('更新完毕')
})
onBeforeUnmount(() => {
  console.log('卸载之前')
})
onUnmounted(() => {
  console.log('卸载完毕')
})
</script>

自定义 hooks ​

  • 什么是hook？—— 本质是一个函数，把setup函数中使用的Composition API进行了封装，类似于vue2.x中的mixin。

  • 自定义hook的优势：复用代码, 让setup中的逻辑更清楚易懂。

示例代码：

  • useSum.ts中内容如下：

    js
    import { ref, onMounted } from 'vue'

export default function () {
  let sum = ref(0)

  const increment = () => {
    sum.value += 1
  }
  const decrement = () => {
    sum.value -= 1
  }
  onMounted(() => {
    increment()
  })

  //向外部暴露数据
  return { sum, increment, decrement }
}

    js
    import {reactive,onMounted} from 'vue'
import axios,{AxiosError} from 'axios'

export default function(){
  let dogList = reactive<string[]>([])

  // 方法
  async function getDog(){
    try {
      // 发请求
      let {data} = await axios.get('https://dog.ceo/api/breed/pembroke/images/random')
      // 维护数据
      dogList.push(data.message)
    } catch (error) {
      // 处理错误
      const err = <AxiosError>error
      console.log(err.message)
    }
  }

  // 挂载钩子
  onMounted(()=>{
    getDog()
  })

  //向外部暴露数据
  return {dogList,getDog}
}
